# Security Policy

## 支持版本

当前仅保证最新主分支（`main`）上的修复与安全更新。

## 漏洞报告

如果你发现安全问题，请不要直接公开创建 Issue。

建议流程：

1. 使用 GitHub 的私密安全报告（Private vulnerability reporting）功能（如果仓库已启用）
2. 如果未启用，请先创建标题带 `[SECURITY]` 的 Issue，并避免披露可复现细节，等待维护者进一步联系

## 报告内容建议

- 影响范围
- 复现步骤
- 风险等级评估
- 临时缓解方案（如有）

## 响应目标

- 72 小时内确认收到
- 在可复现后尽快给出修复计划
